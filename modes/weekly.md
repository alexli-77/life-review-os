# 模式：weekly（周快速模式）

每周执行一次，快速对比上周执行与 OKR，生成下周计划。

## 触发

```
/weekly-review
/weekly-review weekly
```

## 读取范围

- 当前年份 Weekly 文档中的 **OKR 定义段落**（完整）
- 当前年份 Weekly 表格中 **最近 2 周** 的执行列（要务 + retro）
- **不读取**使命宣言和五年计划（由 quarterly 模式负责）

## 用户输入

每次触发时，向用户询问：

```
本周 retro（一两句）：完成了什么，卡在哪里？
本周感受（一句）：精力/情绪/外部压力如何？
```

若用户已在触发命令中提供（如 `/weekly-review retro: xxx 感受: xxx`），直接使用，不再询问。

## 分析重点

- 上周 MIT 是否完成
- 连续未完成的 KR（⭕️ 连续 2 周）
- 本周要务数量是否超过 8 条

## 输出格式

```markdown
## 📊 上周执行（{MM.DD}-{MM.DD}）

✅ 完成
- {要务}
- ...

⭕️ 未完成
- {要务}（{一句话原因}）
- ...

{框架诊断摘要（1-3 句，来自 frameworks/）}

---

## 📋 下周计划（{MM.DD}-{MM.DD}）

**MIT 🔴**：{唯一 MIT}

1. {要务}
2. {要务}
3. {要务}
...（共 6-8 条）

[如有余力]
- {低优先级要务}
```

## 写回行为

`auto_write: true` 时，自动执行 `engine/04-write.md` 的流程，
将计划写入飞书 Weekly 表格的新列，并返回文档链接。

## 时间预期

整个流程（读取 + 分析 + 生成 + 写回）在 **2 分钟内**完成。
