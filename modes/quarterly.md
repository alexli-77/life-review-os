# 模式：quarterly（季末深度复盘）

每季度末执行一次，从使命宣言和五年计划出发，检验方向是否正确。

## 触发

```
/weekly-review quarterly
```

建议在每季度最后一周触发（3月末、6月末、9月末、12月末）。

## 读取范围

比 weekly/biweekly **多读**以下文档：
- 使命宣言（全量）
- 五年计划（只取用户相关章节，如"百岁目标"）
- 当前年份 **全量** OKR 执行数据（所有季度）
- 历史年份 OKR 完成情况摘要（重点看结构性偏移）

## 用户输入

```
本季度 retro（3-5 句）：完成了什么，最大的卡点，感受如何？
这季度最让你有成就感的一件事：
这季度最让你感到焦虑或困扰的一件事：
```

## 核心分析：方向而非执行

quarterly 模式不关注单周执行细节，而是从高度看：

### 1. 使命宣言对齐度

将本季度实际花时间最多的事项，与使命宣言中的角色和价值观对比：
- 高度对齐 → 确认方向正确
- 低度对齐 → 识别是"被生活推着走"还是"主动选择的妥协"

### 2. 长期未突破的方向

识别连续 2+ 季度完成率为 0% 的 OKR：
- 这是策略问题还是方向本就不对？
- 是否应该降级、重构或放弃？

### 3. OKR 数量与容量的匹配

回顾本季度 P0 OKR 数量 vs 实际执行容量：
- 是否每季度都在计划更多但完成更少？
- 是否需要在下季度主动缩减 OKR 数量？

### 4. 五年计划进度

检查与五年计划中当前阶段目标的距离：
- 是否在正轨上？
- 是否有某个方向需要在下季度加速？

## 输出格式

```markdown
## 🧭 季度诊断报告（{年份} Q{N}）

### 使命对齐

{2-3 句，描述本季度行动与使命宣言的对齐程度}

### 完成 ✅
- {OKR/KR}：{简评}
- ...

### 长期停滞 🔴（需决策）
- {OKR}：已连续 {N} 季度未突破
  → 建议：{继续/降级/重构/放弃，附理由}

### 下季度方向校准

**保持**：
- {值得继续投入的方向}

**调整**：
- {需要改变策略的 OKR}：{调整方向}

**放弃/降级**：
- {不再符合当前阶段的 OKR}：{理由}

### 下季度 OKR 建议

{P0 OKR 建议，不超过 4 个；P1 不超过 2 个}

---

> 是否将以上内容写入飞书？（季末报告默认先确认）
```

## 写回行为

`auto_write: false`（默认）：在对话中展示完整报告，等待用户确认后再写回。

用户确认写回后：
- 将季度诊断摘要追加到当年 Weekly 文档末尾（不写入周表格）
- 下季度的第一周计划写入表格新列
