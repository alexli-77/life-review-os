# 分析框架：Stephen Covey 七个习惯

使用柯维框架对计划与执行进行诊断，重点关注"为什么忙却没效果"。

## 核心诊断维度

### 1. P/PC 平衡（产出/产能）

检查 OKR 中产出型任务（交付、发布、完成）与产能型任务（学习、健康、关系维护）的比例。

- P 型任务（产出）过多，PC 型（产能）被挤压 → 提示"杀鹅取卵"风险
- 作息、学习、锻炼等持续未完成 → 红色警示，产能正在下降

### 2. Q2 占比（重要不紧急）

将本周要务分类：

| 象限 | 描述 | 典型要务 |
|---|---|---|
| Q1 紧急+重要 | 危机、截止 | 签证 DDL、工作紧急需求 |
| Q2 重要不紧急 | 核心价值区 | 作品集、语言学习、读书 |
| Q3 紧急不重要 | 别人的紧急 | 临时协助、无关会议 |
| Q4 不紧急不重要 | 浪费 | 无目的刷手机 |

目标：Q2 占比 ≥ 60%。Q1 长期过高 → 提示"一直在救火"。

### 3. MIT 一致性

检查标注为 MIT 的任务是否真的是"这周最重要的一件事"：
- MIT 数量 > 2 → MIT 失效，提示收窄
- MIT 连续未完成 → 可能是方向问题而非执行力问题

### 4. 成熟度锚点

快速判断用户当前在哪个阶段：

| 阶段 | 信号 | 对应建议 |
|---|---|---|
| 依赖期 | 未完成归因于外部（环境、他人、运气） | 拉回影响圈，聚焦可控事项 |
| 独立期 | 有计划、有执行，但效率低或方向偏 | Q2 时间管理、MIT 聚焦 |
| 互赖期 | 计划成熟，卡在协作或关系上 | 双赢协议、移情沟通 |

### 5. 生活中心稳定性

检查 retro 中的情绪触发词，判断是否有生活中心漂移：
- "流量低就焦虑" → 名利中心
- "工作没进展就很差" → 工作中心
- "情绪受伴侣影响大" → 配偶中心

> 原则中心的人：外部波动不影响内在稳定，行动由价值观驱动而非外部反馈。

## 输出模板

在 `engine/02-analyze.md` 的 `framework_diagnosis` 字段中填入以下内容：

```
**柯维诊断摘要**

Q2 占比：{x}%（建议 ≥ 60%）
MIT 完成率：{x}%
本周象限分布：Q1 {n}项 / Q2 {n}项 / Q3 {n}项

主要发现：
- {发现1，一句话，直接点出现象和背后的范式问题}
- {发现2}

本周小实验（具体可操作）：
- {一件本周可以做的小事，验证某个假设或改变某个习惯}
```

## 语气原则

- 先移情，后诊断：承认用户处境的难度，再指出范式问题
- 不说"你应该"，说"数据显示"或"你的 retro 表明"
- 不归咎执行力，优先找结构性原因
- 每次给出一个"本周小实验"，而不是空洞的"你应该更努力"
