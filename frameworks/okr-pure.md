# 分析框架：纯 OKR

不引入外部方法论，纯粹基于 OKR 进度数据进行诊断。适合不熟悉柯维框架、或希望只关注目标完成率的用户。

## 核心诊断维度

### 1. KR 完成率

逐 KR 统计完成情况：
- 完成（✅）
- 进行中（Doing-xx%）
- 未开始（⭕️）
- 长期停滞（连续 2+ 周 ⭕️，标记为 🔴）

### 2. P0 vs P1 执行偏差

检查 P0 OKR 的完成率是否高于 P1：
- P0 完成率低于 P1 → 优先级执行倒置，提示关注
- 所有 P0 都停滞 → 整体执行出现系统性问题

### 3. OKR 范围蔓延

检查周要务是否出现 OKR 之外的新任务：
- 少量 → 正常
- 大量 → OKR 范围蔓延，建议将新任务纳入或明确排除

### 4. 关键结果可衡量性

检查 KR 描述是否有可量化的验收标准：
- ❌ 模糊：`提升设计能力`
- ✅ 可衡量：`完成 3 个 case study 并发布`

若 KR 模糊，提示用户在下次 OKR 更新时修正。

## 输出模板

```
**OKR 诊断摘要**

P0 完成率：{x}%
P1 完成率：{x}%
长期停滞 KR：{列表，如有}

主要发现：
- {发现1}
- {发现2}

下周重点 KR：
- {KR名称}：{原因}
```

## 语气原则

- 数据说话，不做主观判断
- 只报告事实，建议聚焦于"下周优先做什么"
- 不引入柯维术语（Q2、MIT、影响圈等）
