# Engine 03 — 生成下周计划

基于分析结果，生成结构化的下周要务列表。

## 输入

- `engine/02-analyze.md` 的输出结构
- 当前 OKR 定义（优先级、KR 进度）
- 用户感受（影响计划强度）
- 当前模式（weekly / biweekly / quarterly）

## 计划生成规则

### 规则 1：严格限制 MIT 数量

**每周只能有 1 个 MIT**（Most Important Thing）。

MIT 选择标准（按优先级，**有 metadata 时优先用 metadata**）：

#### 有 metadata（vault.enabled = true 且 metadata 存在）

1. `urgency = overdue` 且 priority ∈ {P0, P1}：必须升级为 MIT，否则继续滞后
2. `urgency = this_week` 且 priority = P0 且进度 <50%：强候选 MIT
3. 上周 MIT 未完成且 deadline 未过：延续为本周 MIT
4. `urgency = next_two_weeks` 且 P0：候选 MIT（有提前量）
5. 都不满足时，按规则"P0 中进度最落后的 KR"

> phantom / parked 状态的 KR **永远不进 MIT 候选**，除非用户在 dialog 里显式启用。

#### 无 metadata（fallback，原有逻辑）

1. 上周已标注 MIT 但未完成 → 延续为本周 MIT
2. P0 OKR 中进度最落后的 KR → 提升为 MIT
3. 有明确截止时间压力的 KR（从描述里关键词识别）

若上周有 2+ 个 MIT 且完成率低，本周主动收窄为 1 个，并在输出中说明原因。

### 规则 1.5：Phantom KR 自动剔除（仅有 metadata 时）

`status = phantom` 的 KR 不进入下周要务建议。**第二次连续出现该状态时**，在输出末尾加一段提示：

```
🔍 **下季度建议**
以下 KR 已连续被标记为 phantom（无 deadline 且未启动）：
- O3_KR4: 完成自己的 portfolio（标记于 2026-05-04）
是否在下季度规划时移除或降级？
```

### 规则 2：按 OKR 优先级分配任务量

```
P0 OKR → 每个最多 2 条要务
P1 OKR → 每个最多 1 条要务
长期/家庭类 → 不占周要务槽，除非有具体行动
```

总要务数建议控制在 **6-8 条**（含 MIT）。超过 8 条时主动裁剪低优先级条目。

### 规则 3：要务粒度要"可执行"

- ❌ 太模糊：`推进作品集`
- ✅ 足够具体：`完成作品集个站首页移动端适配并部署`

每条要务应在 1-3 小时内可完成，或有明确的检验标准（✅/⭕️）。

### 规则 4：结合用户感受调整强度

| 用户感受 | 计划调整 |
|---|---|
| 精力充沛、情绪稳定 | 正常计划强度 |
| 情绪低落、外部压力大 | 减少 1-2 条要务，MIT 保留，其余降级 |
| 极度倦怠 | 只保留 MIT，其余全部标记为"如有余力" |
| 生活重大变化（旅行/考试等） | 主动标注该周为"轻量周"，计划减半 |

### 规则 5：渐进自动化原则

初期（V1）：每条要务都基于用户提供的 retro 推导，AI 不凭空生成。
成熟期（V2+）：AI 可根据历史执行模式预判下周要务，用户只需确认或调整。

## 输出格式

生成供写回飞书的结构化计划：

```
MIT 🔴：{要务描述}

KR1（{OKR名称}）：
1. {要务1}  MIT/—
2. {要务2}

KR2（{OKR名称}）：
1. {要务}

KR3（{OKR名称}）：
1. {要务}

[如有余力]
- {低优先级要务}
```

## biweekly / quarterly 模式的额外输出

### biweekly 追加：节奏校正建议
```
## 双周趋势
- {KR} 已连续 {N} 周未完成，建议：{具体调整}
- 本双周 MIT 完成率：{x}%，{高于/低于}上双周
```

### quarterly 追加：方向校准建议
```
## 季度诊断
- 与使命宣言的偏离：{描述}
- 建议降级的 OKR：{列表}
- 建议升级关注的 OKR：{列表}
- 下季度重点：{1-2 句}
```
